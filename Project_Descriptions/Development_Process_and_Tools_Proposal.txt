프로젝트: SentiVision(센티비젼)

1) 문서 목적
- 본 문서는 SentiVision 프로젝트의 개발 과정을 단계별로 정리하고,
  각 단계에서 필요한 핵심 Tool과 사용 목적을 제안하기 위해 작성함.
- 기존 프로젝트 제안서(기능/기술 스택)와 저장소 운영 방식(DORA 지표)을 반영함.

================================================================================

2) 개발 과정 제안 (Process)

[1단계] 요구사항 정의 및 범위 확정
- 사용자 시나리오 정의: 그림 그리기 -> 분석 요청 -> 감정 결과 확인 -> 피드백 제출
- MVP 범위 확정:
  1. iOS 캔버스 드로잉
  2. 색감 추출 + 감정 분류 API
  3. 결과 시각화
  4. 피드백 저장
- 성공 지표 정의:
  - 모델 정확도 목표(초기 80%, 개선 85%+)
  - 응답 시간 목표(예: 분석 요청 2초 이내)

[2단계] 아키텍처 설계
- 모바일(iOS), 백엔드(API), ML 파이프라인, DB를 분리 설계
- 데이터 흐름 설계:
  - 이미지 입력 -> 전처리 -> 색상 추출 -> 감정 분류 -> 결과 반환 -> 피드백 저장
- API 명세서 작성(요청/응답 형식, 오류 코드, 버전 정책)

[3단계] 기능 구현
- iOS: SwiftUI 캔버스, 분석 요청/결과 화면
- Backend: FastAPI 또는 Flask 기반 REST API
- ML: Saliency Map + KMeans + KNN 파이프라인 구현
- DB: 사용자/분석 로그/피드백 데이터 저장 구조 구현

[4단계] 테스트 및 품질 관리
- 단위 테스트(알고리즘, API), 통합 테스트(iOS-API 연동)
- 모델 품질 검증(정확도, 오분류 분석)
- 성능 테스트(처리 속도, 동시 요청 안정성)

[5단계] 배포 및 운영
- iOS TestFlight 배포 후 사용자 테스트
- 백엔드 클라우드 배포(AWS 등)
- 운영 지표 모니터링(DORA + 서비스 로그)

[6단계] 피드백 기반 개선
- 사용자 수정 데이터를 학습 데이터로 축적
- 주기적 재학습 및 성능 비교
- 우선순위 기반 기능 개선(UX, 모델 정확도, 속도)

3) 필요 Tool 제안 (Tooling)

A. 개인 개발/형상관리
1. Git + GitHub
- 브랜치 전략(main/feature)으로 변경 이력 관리
- 기능 단위 커밋 메시지 규칙(예: feat/fix/refactor)으로 변경 목적 추적

2. 개인 작업 관리 도구
- GitHub Issues 또는 로컬 TODO 문서로 작업 목록 관리
- 우선순위(높음/중간/낮음)와 마감일 기준으로 개인 일정 추적

B. 개발 환경
1. iOS
- Xcode + SwiftUI
- Canvas 기반 드로잉 인터페이스 및 결과 시각화 UI 구현

2. Backend/API
- Python + FastAPI(권장) 또는 Flask
- FastAPI 사용 시 문서 자동화(OpenAPI)로 API 점검 및 유지보수 효율 향상

3. ML/데이터
- OpenCV: 이미지 전처리/특징 추출
- scikit-learn: KMeans, KNN 모델 구현
- NumPy/Pandas: 데이터 처리 및 실험 관리

4. 데이터 저장
- PostgreSQL
- 분석 이력, 사용자 피드백, 모델 버전별 결과 저장

C. 품질/테스트
1. pytest
- API/로직 단위 테스트 및 회귀 테스트 자동화

2. (선택) lint/format
- black, isort, ruff 또는 flake8
- 코드 스타일 일관성 및 기본 오류 사전 제거

D. CI/CD 및 지표
1. GitHub Actions
- 테스트 자동 실행
- DORA 지표 수집 자동화(현재 저장소 스크립트 활용)

2. DORA Metrics Script
- scripts/collect_dora_metrics.py
- metrics/dora_metrics.json
- 리드타임, 배포 빈도, MTTR, 변경 실패율 추적

E. 운영/관측
1. Docker Compose
- 로컬에서 API/DB/모니터링 구성 재현

2. Prometheus
- 서비스 메트릭 수집(응답 시간, 요청 수, 에러율 등)

================================================================================

4) 간략 실행 계획 (초안)

Phase 1 (1~2주): 기획/설계
- MVP 기능 확정, 데이터 모델/아키텍처/API 명세서 작성

Phase 2 (3~6주): 핵심 기능 구현
- iOS 캔버스 + Backend API + ML 1차 파이프라인 구현
- 기본 정확도 80% 달성 목표

Phase 3 (7~9주): 통합/테스트
- iOS-Backend 연동 안정화
- 단위/통합 테스트 구축, 성능 병목 제거

Phase 4 (10~12주): 배포/개선
- TestFlight 배포 및 피드백 수집
- 피드백 데이터 반영 재학습, 정확도 85%+ 목표

운영 원칙
- 하루 단위 작업 목표 설정(작게 쪼개서 완료)
- 기능 완료 후 즉시 테스트 및 커밋
- 주간 DORA 지표 점검 및 개인 회고 반영
