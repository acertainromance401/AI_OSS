# PRD: SentiVision

작성일: 2026-03-13  
문서 버전: v1.0

## 1. 제품 개요
SentiVision은 사용자가 캔버스에 그린 그림의 색감 정보를 분석하여 감정 상태를 추정하고, 사용자 피드백을 통해 모델을 점진적으로 개선하는 감성 컴퓨팅 제품이다.

## 2. 문제 정의
- 기존 AI 서비스는 업무 자동화 중심이며 감정 이해 기능이 상대적으로 부족하다.
- 텍스트/음성 기반 감정 분석 대비 색채 기반 감정 분석은 접근성이 높고 시각 창작 경험과 결합하기 쉽다.

## 3. 목표
- 사용자의 그림 색상 팔레트 기반 감정 예측 제공
- 예측 결과를 직관적으로 시각화
- 사용자 피드백 수집으로 모델 정확도 향상

## 4. 비목표 (Out of Scope)
- 임상 진단/의료적 판단 제공
- 실시간 멀티유저 협업 드로잉
- 텍스트/음성 멀티모달 감정 분석 (초기 버전 제외)

## 5. 타겟 사용자
- 개인 사용자: 감정 셀프 체크/기록 목적
- 창작 사용자: 작업물의 정서적 톤 확인 목적
- 연구/학습 사용자: 색상-감정 연관성 실험 목적

## 6. 핵심 사용자 시나리오
1. 사용자는 iOS 앱에서 그림을 그린다.
2. 분석 버튼을 누르면 앱이 색상 정보를 API로 전송한다.
3. API는 대표 색상 기반 감정 점수와 예측 결과를 반환한다.
4. 사용자는 결과를 확인하고, 필요 시 정답 감정을 수정 피드백으로 제출한다.
5. 피드백은 저장되어 다음 모델 개선에 사용된다.

## 7. 기능 요구사항

### FR-1. 드로잉 및 분석 요청
- 앱은 사용자가 그림을 그릴 수 있는 캔버스를 제공해야 한다.
- 분석 요청 시 대표 색상 정보(RGB 팔레트 + 가중치)를 서버로 전송해야 한다.

### FR-2. 감정 분석 API
- 엔드포인트: POST /analyze
- 입력: palette(1~16개 RGB), 선택 입력 weights
- 출력: predicted_emotion, confidence_scores
- 잘못된 입력(예: 음수 weight)은 검증 오류를 반환해야 한다.

### FR-3. 피드백 API
- 엔드포인트: POST /feedback
- 입력: predicted_emotion, corrected_emotion, palette, note(선택)
- 결과: 피드백 저장 성공 여부 반환

### FR-4. 결과 시각화
- 앱은 예측 감정과 감정별 점수를 그래프 또는 카드 형태로 표시해야 한다.

### FR-5. 데이터 기반 감정 매핑
- 서버는 RGB-감정 데이터셋을 기반으로 감정 점수를 계산해야 한다.
- 색상 연관 감정 보정(연관어/온기 정보)을 반영해야 한다.

## 8. 비기능 요구사항
- 성능: 단일 분석 요청 평균 응답 2초 이내 목표
- 안정성: API health endpoint 제공 (GET /health)
- 보안:
  - 민감정보를 코드에 하드코딩하지 않는다.
  - 환경변수(.env) 기반 설정을 사용한다.
- 컴플라이언스:
  - 데이터셋 재배포 전 라이선스 확인 필수
  - 미확인 데이터는 공개 저장소에 커밋하지 않는다.

## 9. 현재 기술/시스템 범위
- Frontend: Swift/SwiftUI (예정)
- Backend: FastAPI
- ML/데이터: 색상 기반 매핑 로직 + CSV 데이터셋
- 테스트: pytest 기반 API 테스트

## 10. 성공 지표 (KPI)
- 분석 정확도: 1차 80%+, 목표 85%+
- 사용자 피드백 응답률: 분석 요청 대비 피드백 제출 비율
- API 안정성: /health 성공률 99% 이상 (운영 환경 기준)

## 11. 마일스톤
- M1 (1~4주): 아키텍처 설계, 데이터셋 정리, API 골격 구축
- M2 (5~8주): 핵심 분석 파이프라인 구현, API 연동
- M3 (9~12주): 통합 테스트, 결과 시각화, 피드백 루프 완성
- M4 (13~16주): 성능 최적화, 재학습, 배포 준비

## 12. 리스크 및 대응
- 데이터 라이선스 미확정
  - 대응: 출처/라이선스 문서화 후 공개 반영
- 감정 분류 정확도 부족
  - 대응: 피드백 데이터 축적 및 주기적 재학습
- 플랫폼 통합 지연(iOS-API)
  - 대응: API 계약 우선 고정, Mock 데이터로 병행 개발

## 13. 수용 기준 (Acceptance Criteria)
- API가 /analyze, /feedback, /health를 정상 제공한다.
- 테스트 코드가 핵심 시나리오(정상/검증실패/저장)를 통과한다.
- README에 실행 방법과 보안/데이터 정책이 명시된다.
- 데이터 라이선스 문서가 존재하고 상태가 추적된다.
