API 테스트 실행 가이드
프로젝트: SentiVision FastAPI Backend
작성일: 2026-03-13

1) 테스트 코드 위치
- tests/test_api.py

포함된 테스트
- /health 엔드포인트 상태 확인
- /analyze 응답 구조 및 점수 합 검증
- /analyze 음수 weight 입력 시 검증 에러(422) 확인
- /feedback 저장 동작 및 CSV 생성 확인

================================================================================

2) 사전 준비

권장 Python 버전
- 3.11 (권장), 3.12 또는 3.13
- 3.14에서는 일부 의존성(pydantic-core) 빌드 오류가 발생할 수 있음

(1) 가상환경 생성/활성화
- macOS/Linux:
  python3.11 -m venv .venv
  source .venv/bin/activate

(2) 패키지 설치
- pip install -r requirements.txt

================================================================================

3) 테스트 실행 방법

(1) 전체 테스트 실행
- pytest -v

(2) 특정 파일만 실행
- pytest -v tests/test_api.py

(3) 특정 테스트 함수만 실행
- pytest -v tests/test_api.py::test_analyze_endpoint_returns_prediction_and_scores

================================================================================

4) 실행 결과 해석

- PASSED: 테스트 통과
- FAILED: 코드 동작 불일치 또는 데이터/환경 문제

주의 사항
- 테스트 코드는 임시 샘플 CSV를 테스트 런타임에 자동 생성함.
- 따라서 공개 저장소에 data/*.csv가 없어도 테스트 실행 가능.

================================================================================

5) 자주 발생하는 문제와 해결

문제 1) ModuleNotFoundError: fastapi / pydantic / httpx
- 원인: 의존성 미설치
- 해결: pip install -r requirements.txt

문제 2) 테스트에서 CSV 관련 예외 발생
- 원인: 테스트 중 임시 데이터 생성 실패(권한/경로 문제)
- 해결: pytest를 프로젝트 루트에서 실행하고 쓰기 권한 확인

문제 3) pytest 명령을 찾지 못함
- 원인: 가상환경 비활성화 또는 pytest 미설치
- 해결:
  1) source .venv/bin/activate
  2) pip install -r requirements.txt

문제 4) pydantic-core 빌드 실패 (PyO3 최대 버전 관련)
- 원인: Python 3.14 환경에서 빌드 시도
- 해결: Python 3.11로 가상환경 재생성 후 설치
  1) rm -rf .venv
  2) python3.11 -m venv .venv
  3) source .venv/bin/activate
  4) pip install -r requirements.txt
